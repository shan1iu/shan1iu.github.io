useState

1

const [state, setState] = useState(initialValue);
  • 与类组件的this.setState不同，在hooks中更新state不会做合并操作，只会整个替换。
  • 利用了闭包

useEffect

  • componentDidMount + componentDidUpdate + componentWillUnmount.
  • 每次调用render()，替换之前的effect
  • 区别于componentDidMount or componentDidUpdate,effects不会阻止浏览器更新显示。在一些不常见的场景中（比如计算布局），可以使用useLayoutEffect.

  • 在下次运行effects之前清除effects.

    1
2
3
4
5
6

    useEffect(()=>{
// componentDidMount + componentDidUpdate
return ()=>{
// componentWillUnmount
}
},[])

useRef

1
2
3

const inputRef = useRef();
// access with inputRef.current
<input ref={inputRef}/>
  • 比类组件的ref更强大，可用于保存任何的可变的值（通过传入参数）。useRef()创建一个对象，与自己创建{current: ...}不同的是useRef()在每次调用render的时候都返回同样的ref对象。

  • Use Case：配合`useRef检查setState之前组件是否已卸载

    1
2
3
4
5
6

    const isExist = useRef(true);
useEffect(() => {
return () => {
isExist.current = false;
}
}, []);

useLayoutEffect

  • 与useEffect相同, 在DOM操作之后同步执行

useCallback

  • 不会立刻调用
  • 避免函数方法多次被创建

  • 缓存方法本身

    1
2
3

    const callBack = useCallback(() => {
setState()
}, [setState]);

  • Use Case：在forLoop中给每个子item绑定事件

    {
array.map((item, index) => {
return <Child callBack={callBack} key={index}/>

}

useMemo

1

const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
  • 立刻调用
  • 用于优化计算属性
  • 缓存保存的结果
